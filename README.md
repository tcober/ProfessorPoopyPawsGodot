# Professor Poopy Paws

A Zelda: *A Link to the Past*–style action-RPG about a brilliant science cat who — after
a public humiliation, a devastating loss, and the draining of the world's magic — is
pulled out of hermithood to restore magic to the world and find love again. Tonally it's
**Adventure Time** meets **Final Fantasy**: whimsical and absurd on the surface, earnest
underneath.

Built in **Godot 4.6** (GDScript, GL Compatibility renderer) with an art style chasing
the 16-bit greats: Chrono Trigger's pixel density, Secret of Mana's lush action-RPG
feel, and Paper Girls' surreal duotone color scripting, all inside a
steampunk-inflected medieval fantasy world — brass and flasks, candles and gears.

> The full design bible lives in [docs/DESIGN.md](docs/DESIGN.md) — story, themes,
> influences, asset specs, and conventions. It's the single source of truth.

## The game right now

The current build is a **combat-first cut**: a small, dense slice of the game used to
hone the battling and the look.

- You play **Fuji**, a tortoiseshell librarian cat with round brass reading glasses, a
  plum scholar's robe, and a tome hugged to her chest.
- The game boots into a loft bedroom, down the stairs through a kitchen/steampunk-lab
  great room, and out the front door onto a **Chrono Trigger–style tiled overworld** —
  towns are drawn cluster icons, landmarks anchor the horizon, and one winding trail
  leads NE to **Whisker Meadow**, where slimes await.
- Combat is real-time top-down: overhead tome slams up close, blow-pipe darts at range.
  Slimes pop in two hits and respawn elsewhere in the meadow.

A second playable character, **Basil** (the professor himself, with an instant-fire
laser gun, recoil skid, and beaker magazines), is fully working but parked — the scenes
are player-agnostic, so swapping is a one-line change.

## Running it

Open the project in **Godot 4.6** and press Play, or from the repo root:

```sh
godot --path .
```

The dev window runs at 1152×648 (3× the base 384×216 resolution).

### Controls

| Key | Action |
| --- | --- |
| **WASD** / arrows | Move (8-way) |
| **J** / Space | Attack — tome swing (Fuji) / laser (Basil) |
| **K** / Shift | Hop — air-steerable dodge |
| **L** | Blow-pipe dart |
| **R** | Reload (Basil's beaker magazines) |
| **E** | Interact |

## How it's built

- **Component-based architecture** — reusable behavior lives in
  [components/](components/) (health, hitbox, hurtbox); entities in
  [entities/](entities/) compose them; shared data are `Resource`s in
  [resources/](resources/); rooms and levels in [scene/](scene/).
- **One scene pipeline, one map format** — every scene (interiors, overworld, town,
  meadow) is a plain-text tile map driven through a shared Python tile kit
  ([assets/_tilekit.py](assets/_tilekit.py)). Move a feature character in the map txt
  and it moves in-game. A new scene is just a map txt plus a thin config script.
- **Procedurally authored pixel art** — all tilesets, sprites, and FX are generated by
  the Python scripts in [assets/](assets/), sliced and deduped into real Godot
  TileSets. Two TileMapLayers (under/over entities) let bodies walk behind railings,
  lintels, and rooflines; collision is built at runtime from the same map file.

### Regenerating art

```sh
python3 assets/_gen_tileset_overworld.py   # (or any other _gen_*.py)
python3 assets/_check_art.py               # sanity-check the output
godot --headless --import                  # re-import — required, or atlases render scrambled
```

Scenes can be eyeballed without launching the game via `tools/shot.gd` (see "Art
pipeline" in [docs/DESIGN.md](docs/DESIGN.md)).
